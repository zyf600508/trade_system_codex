# 系统目标
获取股票历史数据，通过持续监控某一只股票的每日资金流向，成交量，和每日筹码集中度变化情况，判断某一只股票是否有主力持续关注，主力资金持续流入，筹码集中度不断提高，主力持续吸筹，抓住主力的持续吸筹，主力还未拉升阶段的买入机会。

# 核心参考的指标
- 筹码集中度和筹码分布，判断主力低位吸筹或者高位出货
- 股价处于高位还是低位
- 主力资金流向
- 成交量

# 需求
1、监控的股票池范围
- 包含上交所，深交所，创业板，科创板的股票
- 排除 st 和 *st 的股票
- 排除上市不到半年的股票
- 排除市值超过5000亿元的股票
2、数据准备
- 首次运行下载符合1部分的全部股票的过去一年的历史数据，数据存入数据库。数据库设计要参考需要下载哪些数据指标。
- 所有需要的历史数据，尽量提前下载存储数据库。如果有些数据无法提前下载，有需要开盘实时下载
- 每日收盘后自动下载当天的数据，更新到数据库
- 数据下载的数据库源可以使用多个数据源，防止某一个数据源不稳定，切换成另外的数据源
3、监控和选股买入
- 总体原则是通过资金流向，成交量和筹码分布，发现主力持续吸筹买入，成交量逐渐放大，筹码集中度越来越集中的主拉伸之前的买入机会
- 通过计算某一只股票的整体分数，来判断操作建议：重点观望，建议买入，强烈建议买入，卖出，强烈建议卖出等
- 股票股价处于相对低位加分，股票股价处于相对高位减分
- 股票股价处于相对低位并且在低位横盘较长时间加分
- 股票股价处于相对低位并且在低位横盘较长时间，然后在低位逐渐温和放量，伴随主力资金持续流入和筹码集中度逐渐升高加分
- 股票股价处于相对低位并且在低位横盘较长时间，主力资金流入伴随放量拉升加分
- 根据以上原则，制定需要下载的指标参数：主力净流入，小单净流入，大单净流入，超大单净流入，连续n天主力净流入，当天成交量，主力吸筹，筹码集中度持续升高，筹码平均成本，筹码价格区间等。
- 操作建议按评分排序，最强烈建议的排在最前
- 你可自行分析这个策略，可以考虑添加相关的策略和指标，从而达到更高的准确度和胜率
- 设置一个评分标准，来判断重点观望，建议买入，强烈建议买入，卖出，强烈建议卖出等操作。评分标准放在配置文件里，方便随时调整
4、监控和选股卖出
- 总体原则是通过资金流向，成交量和筹码分布，发现主力持续流出，筹码越集中度来越分散，判断主力在高位出货，应该果断卖出
- 止盈卖出：盈利 30%
- 止损卖出：亏损 15%
5、持仓管理
- 初始资金 10万
- 最多持有 5只股票
- 每次买入和卖出需根据现在股价处于高位还是低位，当前仓位，买入评分结果，综合得出买入的数量
6、其他
-  所有配置的参数值放到统一的配置文件里，方便随时查看和修改，每个参数要有注释
-  所有过程要有日志信息记录到日志文件和数据库
-  所有注释和日志输出都是中文
-  配套输出系统的执行策略，使用方式说明
